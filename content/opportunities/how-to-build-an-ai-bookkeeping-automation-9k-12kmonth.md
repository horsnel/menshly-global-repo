## The Workflow: Step-by-Step With Every Shortcut

### The Workflow: Step‑by‑step With Every Shortcut  

The whole thing boils down to a tight loop of **import → label → reconcile → report → deliver**.  
Here’s the exact flow, the tools you plug in, and the tiny tricks that shave hours off the math.

---

#### ### Step 1: Data Ingestion (15 min)

Grab every bank, credit‑card, and vendor file in one go.  
Use **Zapier** to hook your bank’s API or a CSV uploader into **Make.com**.  
Zapier’s free tier gives you 100 tasks/month; that’s enough for 2–3 clients a day.  
In Make.com, set up a scenario that watches the Zapier folder, parses the file, and pushes each transaction into a Google Sheet (or a Notion table for that quick “no‑code” feel).  

**Prompt for Make.com**  
```
When a new file appears:
  • Read CSV
  • Loop rows
  • Output JSON to Google Sheets
```  

**HACK:**  
*If your bank uploads in PDF, run the PDF through the free “PDF to Text” module in Make.com, then feed the text to a GPT prompt that pulls out the amount, date, and merchant.*  

---

#### ### Step 2: AI Classification & Reconciliation (30 min)

Now you’ve got raw numbers. Time to tell them what they are.  
Spin a small script in **Replit** (free tier, $7/month for a Pro account) that calls **ChatGPT** via the OpenAI API.  
Feed each transaction a prompt like:

```
Classify this transaction: “Payment to Office Depot for office supplies – $127.45”  
Categories: Office Supplies, Utilities, Travel, Other.  
Return the category with confidence score. 
```  

Set the temperature to 0.2 for consistency.  
Use a 1,000‑token limit per call; you’ll hit the free tier’s 60k tokens/month limit after 60 calls.  

Once classified, push the data into a **Notion** database.  
Notion’s free plan handles 1,000 blocks, so you’re fine until you hit 10 clients.  

**HACK:**  
*Batch the prompts: send 50 rows at once to reduce API calls. ChatGPT will still return a list of results with the same accuracy.*

---

#### ### Step 3: Reporting & Insights (20 min)

Clients want clean visuals, not a spreadsheet dump.  
Use **Canva** (free tier, upgrade to $12/month Pro for brand kit) to pull in the categorized data via an API‑to‑CSV export.  
Create a one‑page “Financial Snapshot” that shows:
- Total spend by category
- Monthly trend line
- Highlight any anomalies (e.g., a $5,000 hit to “Travel” in a quiet month)

For the anomaly detection, feed the same data to a quick **Python** script on Replit that flags anything > 2 × average monthly spend.  

**HACK:**  
*In Canva, use the “Chart” element and link it to a Google Sheet that auto‑updates every 5 min via Make.com. No manual re‑uploading.*

---

#### ### Step 4: Automation & Client Delivery (25 min)

Now you’ve got a report. Time to ship.  
Set up a **Make.com** scenario that takes the Canva PDF, emails it through **Klaviyo** (free tier covers 500 contacts, $20/month after that), and posts a Slack message to your “Clients” channel.  
Add a voice note generated by [**ElevenLabs**](https://elevenlabs.io/) ($15/month for 10k characters) that says: “Hey [Client], your Q1 snapshot is ready. Check your inbox.”  

Use **Calendly** to schedule a quick 15‑min walk‑through. The booking link auto‑creates a Zoom invite (Zoom free tier is fine for 100 min).  
Finally, push a post‑delivery survey via [**Beehiiv**](https://beehiiv.com/) (free for 5,000 subscribers) to capture feedback.
